# Loop v1.4.1

Loop 1.4.1 is a focused performance and execution-reliability update.
Terminals, streamed code responses, and large files stay responsive, while
Loop now validates more of the web-to-daemon boundary and lets the daemon make
the final decisions for commands it executes.

## Faster hot surfaces

- Project terminals now prefer WebGL rendering in the packaged Tauri app, with
  automatic DOM fallback when WebGL is unavailable or its context is lost.
- Streamed code blocks show incoming text immediately and limit
  syntax-highlighting work to roughly five refreshes per second. The completed
  response receives the same exact final highlight as a non-streamed message.
- Text files larger than 2 MiB or 100,000 lines open immediately in a read-only
  plain-text preview instead of mounting Monaco.

## More reliable execution

- The daemon now owns temporary worktree branch names and project-terminal
  working directories and environment variables, then returns the executed
  values to the UI.
- Settings remain optimistic for responsiveness but reconcile to the complete
  state persisted by the daemon.
- Provider model aliases and effort capabilities come from the daemon's model
  catalog, and custom-model limits are enforced server-side.
- Claude ultrathink prompting is applied once at the daemon boundary instead of
  independently in the web app and daemon.
- GitHub Enterprise and self-hosted GitLab remotes are recognized in Git status
  and actions instead of appearing as unknown hosts.

## Stronger contracts

- Native orchestration commands and Build configuration are decoded into typed
  Rust structures, with malformed or unsupported inputs rejected cleanly.
- An automated drift gate compares 14 shared schemas and 23 native RPC methods
  across TypeScript and Rust.
- Development builds validate daemon responses and events at the client
  boundary while avoiding repeated validation of high-frequency streaming
  deltas.
- Activity tones, kinds, and session statuses now use the same bounded
  vocabulary on both sides of the protocol.

## Requirements

- Loop v1.4.1 requires macOS 15 or newer on Apple silicon.
- At least one authenticated local provider runtime is required: Claude Code,
  Codex, or Cursor Agent with ACP support.
- The stable public update feed remains compatible with both older Electron
  installations and current Tauri installations.
