# Loop v1.3.0

Loop 1.3.0 is the first stable Tauri-backed desktop release and a broad workflow
release across the app. It moves Loop to a bundled native backend, keeps the
public macOS update feed compatible with older Electron builds, and improves the
daily coding surface across chat, preview, terminal, Git, project navigation,
settings, and release packaging.

## Highlights

- Desktop runtime moved from Electron/server plumbing to a Tauri shell with a
  native backend bundled as a sidecar.
- The native backend now handles auth, Git, terminal transport, orchestration
  snapshots, project and thread commands, local server discovery, provider
  routing, editor reads, and workspace access.
- Chat and composer flows are steadier, with desktop image file drops,
  preserved pending input during submission, cleaner provider/model state,
  improved user-input panels, and split Markdown rendering.
- Project navigation and sidebar workflows were reworked with project
  workspace state, live project context, local server management, and clearer
  branch/status presentation.
- Preview support now includes desktop preview sessions, native preview
  screenshots, better preview URL resolution, scoped local-server discovery,
  and more resilient preview hooks.
- Git workflows now include branch merge dialogs, Merge PR actions, generated
  commit and PR text using the configured model, clearer pending-branch state,
  and improved Git handoff.
- Editor, terminal, and theme behavior are more reliable, including login-shell
  environment sync, stable terminal sessions, resilient editor loading, shared
  theme tokens, and packaged-build styling fixes.
- Settings and update UX were tightened with restored Cursor provider support,
  stronger provider-auth fallback, native update handling, app-menu/update
  entry points, and release prompts that wait for real update state.
- Release packaging now produces the stable macOS assets needed by both the
  legacy Electron updater and the Tauri updater.

## Shipping notes

- Loop v1.3.0 is macOS arm64-only.
- This is the stable update target for older Electron builds and current Tauri
  builds.
- At least one authenticated local provider runtime is required.
